El proyecto Heirloom mantiene herramientas y utilidades clásicas de UNIX para sistemas operativos modernos de código abierto.

Las utilidades proporcionadas por los paquetes correspondientes se instalan en /usr/heirloom/bin/ y /usr/heirloom/lib/. Algunas utilidades POSIX se proporcionan adicionalmente en /usr/heirloom/bin/posix/.

Nombre Paquetes Notas
Toolchest heirloom-cvsAUR[enlace roto: package not found] Binarios clásicos de UNIX correspondientes a GNU coreutils, diffutils, tar y más, organizados de acuerdo con los diferentes lanzamientos de UNIX.
Bourne shell heirloom-shAUR, heirloom-sh-cvsAUR[enlace roto: package not found]
Herramientas de desarrollo heirloom-devtools-cvsAUR[enlace roto: package not found] Herramientas como yacc, lex, m4, make y sccs. Este paquete, junto con el shell Bourne, forma una base para el resto de los paquetes Heirloom. El uso de los devtools clásicos garantiza un comportamiento consistente, independientemente de los cambios realizados en las herramientas GNU correspondientes.
Herramientas de documentación heirloom-doctoolsAUR, heirloom-doctools-cvsAUR[enlace roto: package not found], heirloom-doctools-gitAUR Una alternativa a groff.
mailx heirloom-mailx-cvsAUR[enlace roto: package not found]
Herramientas de empaquetado heirloom-pkgtools-cvsAUR[enlace roto: package not found] Un port del sistema de gestión de paquetes de Sun SVR4, que se lanzó como código abierto con OpenSolaris.
